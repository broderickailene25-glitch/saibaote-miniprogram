## 五、设计规范（2026-06-04 改为喜茶极简现代风）

### 设计理念

**喜茶新版极简现代风 — 纯白背景 + 黑白灰主色 + 品牌绿仅点缀**

> 参考案例：喜茶新版小程序（16张截图+分析报告存档于图片分析/batch-002-喜茶小程序设计/）

### 视觉风格

| 元素 | 规范 |
|------|------|
| **背景** | 纯白 #FFFFFF，不用任何彩色背景 |
| **主色** | 黑白灰，文字/边框/阴影全部中性色 |
| **品牌绿** | 仅用于：主按钮、Tab选中态、价格高亮 |
| **渐变** | 零渐变，按钮纯色填充 |
| **纹理** | 零纹理，不用宣纸/水墨/SVG背景 |
| **阴影** | 极轻阴影（0.03 透明度），几乎看不见 |
| **留白** | 大量留白（间距 96rpx+），呼吸感 |
| **字重** | 标题 500，正文 400，不用 600+ |
| **字体** | 系统字体：PingFang SC / SF Pro |
| **圆角** | 大圆角（16-32rpx），柔和感 |

### 色彩系统

| 变量 | 色值 | 用途 |
|------|------|------|
| `--bg` | #FFFFFF | 页面背景 |
| `--text-primary` | #1A1A1A | 标题、正文 |
| `--text-secondary` | #666666 | 辅助文字 |
| `--text-tertiary` | #999999 | 次要文字 |
| `--text-placeholder` | #CCCCCC | 占位符 |
| `--border` | #EEEEEE | 边框 |
| `--border-light` | #F5F5F5 | 轻边框 |
| `--green-mid` | #4A7C59 | 品牌绿（仅按钮/选中态） |
| `--shadow-card` | 0 1rpx 4rpx rgba(0,0,0,0.03) | 极轻阴影 |

### 设计原则

1. **纯白背景**：页面背景 #FFFFFF，不用任何彩色背景
2. **黑白灰主色**：文字/边框/阴影全部中性色
3. **品牌色克制**：绿色仅用于关键交互元素（按钮、选中态）
4. **零渐变零纹理**：按钮纯色填充，不用任何装饰纹理
5. **大留白**：模块间距 96rpx+，呼吸感充足
6. **去边框用阴影**：卡片不用边框，用极轻阴影区分层级

### 动效要求

| 要求 | 参考 |
|------|------|
| **丝滑流畅** | 苹果官网级别的过渡动画 |
| **轻盈微交互** | 按钮点击反馈、卡片展开 |
| **性能** | 60fps，不卡顿 |

### 交互原则

- **简洁**：不超过 3 步完成核心操作
- **直觉**：用户一看就知道怎么用
- **优雅**：每个交互都有细节打磨

### 图片资源：腾讯云对象存储（COS）

项目图片统一按腾讯云对象存储形式管理，减少小程序包体积，并让首页、产品、资讯等图片可以独立替换。

| 项目 | 配置 |
|------|------|
| COS 控制台入口 | https://cloud.tencent.com/login?s_url=https%3A%2F%2Fconsole.cloud.tencent.com%2Fcos5 |
| 当前图片域名 | `https://saibaote-images-1410640865.cos.ap-guangzhou.myqcloud.com` |
| 代码入口 | `utils/cos.js` |
| 图片引用方式 | 使用 `cosUrl('目录/文件名.jpg')` 或 `COS_ASSETS.xxx` |

#### COS 图片使用步骤

1. 登录腾讯云 COS 控制台：打开 `https://cloud.tencent.com/login?s_url=https%3A%2F%2Fconsole.cloud.tencent.com%2Fcos5`。
2. 进入图片存储桶：选择赛宝特项目使用的 COS Bucket。
3. 按业务目录上传图片，建议目录如下：
   - `branding/home/`：首页 Hero、品牌展示图。
   - `branding/products/`：产品页品牌卡片图。
   - `products/`：商品图、产品详情图。
   - `news/`：资讯封面图。
4. 上传图片后复制对象路径，不要在页面里硬编码完整 URL。
5. 在 `utils/cos.js` 中统一维护域名和公共图片路径，例如：

```js
const COS_BASE = 'https://saibaote-images-1410640865.cos.ap-guangzhou.myqcloud.com'

function cosUrl(path) {
  return `${COS_BASE}/${path}`
}
```

6. 页面或数据文件里通过工具函数引用图片，例如：

```js
cover: cosUrl('news/news-lixia.jpg')
image: cosUrl('products/product-balcony.jpg')
```

7. 如果新增常用品牌图，放入 `COS_ASSETS`，页面直接使用语义化字段，避免分散维护：

```js
const COS_ASSETS = {
  heroProduct: cosUrl('branding/home/hero-product.jpg')
}
```

8. 发布前检查：
   - 确认 COS 对象路径和代码引用路径完全一致。
   - 确认小程序后台已配置 COS 域名为合法下载域名。
   - 确认本地图片只保留必要占位图，业务大图优先走 COS。
   - 确认微信开发者工具预览时图片能正常加载。

## 十、注意事项

### 开发原则

1. **先做核心功能**：产品展示 + 下单购买优先
2. **设计先行**：每个页面先出设计稿再开发
3. **动效适度**：丝滑但不花哨，不影响性能
4. **移动优先**：所有页面优先适配手机屏幕

### 微信小程序限制

- 包大小限制：主包 2MB，分包 20MB
- 页面栈限制：最多 10 层
- 支付需要企业资质


### 🔄 跨会话恢复流程

> 无论开启多少新会话，只要 README 在，备份就能恢复。

**恢复步骤：**
1. 说 `读核心` 或 `继续赛宝特小程序开发，去读readme`
2. 到 README 的「版本存档系统」找到目标版本
3. 看 `_versions/版本名/MANIFEST.md` 确认要不要恢复
4. 执行 MANIFEST.md 里的恢复命令（cp 覆盖即可）

**创建新版本（以后修改页面时）：**
```bash
# 1. 在 _versions/ 下创建版本目录
mkdir -p _versions/页面名-版本号/

# 2. 复制改动的文件进去
cp pages/xxx/xxx.wxml _versions/页面名-版本号/

# 3. 写 MANIFEST.md
# 4. 在 README 末尾追加版本记录
```
